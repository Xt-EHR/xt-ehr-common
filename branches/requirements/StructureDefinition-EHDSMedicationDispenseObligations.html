Logical Model: Medication dispense model obligations

Official URL: https://www.xt-ehr.eu/specifications/fhir/StructureDefinition/EHDSMedicationDispenseObligations Version: 0.1.0
Draft as of 2025-04-17 Computable Name: EHDSMedicationDispenseObligations

Obligations for the logical model for medication dispense.

Usage:

  • This Logical Model is not used by any profiles in this Implementation Guide

EHDSMedicationDispense 0..* EHDSMedicationDispense
Elements defined in Ancestors:header, dispensingOrganizationOrDevice[x], dispenseLocation, receiver[x], relatedRequest, medication, dispensedQuantity, timeOfDispensation, substitution, dosageInstructions, comment
Medication dispensation model
Instances of this logical model can be the target of a Reference
header 1..1 EHDSMedicationDispenseHeader Header level metadata about the dispense
subject O 1..1 EHDSPatient Patient who the medication was prescribed to. [Used for searching]
ObligationsActor
SHALL:able-to-populate Dispensing System
authorship 1..* Base Authorship
author O 1..1 EHDSHealthProfessional The person who dispensed the product, and takes the responsibility of the dispensation or declining the dispense. [Used for searching]
ObligationsActor
SHALL:able-to-populate Dispensing System
datetime O 1..1 dateTime The time of recording the dispense or declining the request. [Used for searching]
ObligationsActor
SHALL:able-to-populate Dispensing System
lastUpdate 0..1 dateTime Date and time of the last update to the resource
status O 1..1 CodeableConcept Status of the dispense. In case of declining a dispense, the status should be 'declined'. [Used for searching]
ObligationsActor
SHALL:able-to-populate Dispensing System
statusReason[x] O 0..1 Reason for the current status of dispensation, for example the reason why the dispensation was made invalid
ObligationsActor
SHOULD:able-to-populate Dispensing System
statusReasonCodeableConcept CodeableConcept
statusReasonString string
language 0..1 CodeableConcept Language
Binding Description (No ValueSet): (preferred): BCP 47

version 0..1 string Version
identifier O 1..* Identifier Business identifier(s) for the dispense. [Used for searching]
ObligationsActor
SHOULD:able-to-populate Dispensing System
dispensingOrganizationOrDevice[x] 0..* Dispensing organization or device
dispensingOrganizationOrDeviceEHDSOrganization EHDSOrganization
dispensingOrganizationOrDeviceEHDSDevice EHDSDevice
dispenseLocation O 0..1 EHDSLocation Location of dispense
ObligationsActor
SHOULD:able-to-populate Dispensing System
receiver[x] O 0..1 Identification of the person who received the dispensed medication, especially when it was not the patient
ObligationsActor
SHOULD:able-to-populate Dispensing System
receiverEHDSPatient EHDSPatient
receiverEHDSHealthProfessional EHDSHealthProfessional
receiverEHDSRelatedPerson EHDSRelatedPerson
relatedRequest O 0..* Identifier Identifier of the prescription or prescription item the dispense is related to
ObligationsActor
SHOULD:able-to-populate Dispensing System
medication O 1..1 EHDSMedication Exact dispensed product
ObligationsActor
SHALL:able-to-populate Dispensing System
dispensedQuantity O 1..1 Quantity Number of dispensed packages if the pack size is known, or number of smaller items/units
ObligationsActor
SHALL:able-to-populate Dispensing System
timeOfDispensation O 1..1 dateTime Date and time of dispensation
ObligationsActor
SHALL:able-to-populate Dispensing System
substitution O 0..1 Base Indicated whether substitution was made by the dispenser
ObligationsActor
SHALL:able-to-populate Dispensing System
substitutionOccurred O 1..1 boolean Indicated whether substitution was made by the dispenser
ObligationsActor
SHALL:able-to-populate Dispensing System
type O 0..1 CodeableConcept What kind of substitution was made by the dispenser
ObligationsActor
SHALL:able-to-populate Dispensing System
reason O 0..* CodeableConcept Reason why the substitution was made
ObligationsActor
SHALL:able-to-populate Dispensing System
dosageInstructions O 0..* EHDSDosaging Dosaging and administration instructions
ObligationsActor
SHOULD:able-to-populate Dispensing System
comment O 0..* string Additional information or comments

SHOULD:able-to-populate Dispensing System
